VICOScope Help Documentation

General help for VICOScope.

Project Explorer

The project explorer is the main window for managing all projects within VICOScope. Although it is not possible to operate several devices in parallel, you can still operate all connected devices sequentially via the explorer.

How to create a new project?

  • To create a new project you can either use a mouse right-click>>[New Project] within an empty space inside the explorer or use the menu bar: [File]>>[New]>>[Project].
  • The "New VICOProject Wizard" is starting:
    • A meaningful name for the project can be assigned within "Project Name".
    • After clicking on [Browse], the storage location of the project including all future spectrum data can be defined.
    • In order to assign a device to the project, you must first scan for connected devices (if the desired device is not already displayed.).
    • Clicking on [Scan] opens the "Device Selection" window:
      • Select or deselect the desired interface and define "Network/Prefix" for UDP and "IP Range" for TCP, if required. The default settings (all checkboxes enabled; UDP: Network = 192.168.0.0, Prefix = 24; TCP: IP Range = 192.168.0.2 - 192.168.0.2) can be retained when the device is put into operation for the first time.
      • The device search is started by clicking on [Start].
      • Depending on the selected settings, the search process can take several seconds.
      • Selecting a device and clicking on [OK] will assign the device to the project. Note: It is also possible to create an "offline" project without choosing a device.
      • After clicking on [Create], the wizard is finished and the new project is displayed in the explorer.

The explorer is grouped into three levels, which are now explained individually:

  • Project
    • Via mouse right-click>>[Open Directory] the location of the project file, including all spectrum data, is shown in the file explorer of the operating system.
    • The currently active project, for which the digital pulse processor (DPP) parameters are also displayed within the configuration view, is shown in bold. An inactive project can be activated with a mouse double left-click or via mouse right-click>>[Set to Active].
    • If a device is assigned, the serial number is displayed in square brackets after the project name. If this is not the case, "offline" is shown in square brackets.
    • The device of the project can be changed via mouse right-click>>[Device] if several devices are connected. In addition, the device selection can be run through again by mouse right-click>>[Device]>>[Scan].
    • The diagnosis window can be opened with a mouse right-click>>[Diagnosis]. It provides an overview of various static and dynamic device information.
      • The [Refresh] button updates all information once.
      • The switch can be used to activate an auto-refresh mode, which updates all information once a second.
      • The [Export] button can be used to save the displayed information as a JSON file. This might be helpful in support cases.
      • The detector temperature can also be changed within the diagnosis window. A new target temperature must be entered under "SDD Temp/°C" and confirmed with [Apply]. The new target temperature is immediately used for control. After a refresh, the new set temperature is displayed.
    • The project and the associated folder on the host system can be renamed via mouse right-click>>[Rename].
    • The project can be closed by mouse right-click>>[Close], or even completely deleted from the host system after confirmation.
  • Configuration
    • All spectra measured with the same DPP configuration are grouped. Each group has a suffix number as a unique identifier that is displayed after an underscore.
    • Double-clicking on a configuration of any project automatically loads the associated DPP parameters. The next recorded spectrum is then saved in this configuration group within the active project.
  • Spectrum
    • Each spectrum is saved within the project folder as a *.vspc file with the time stamp as the default name in the format "YYYYMMDD-HHMMSS.vspc".
    • In addition to the spectrum raw data and the DPP parameters, the file also contains further information on the measurement within the JSON structure. This includes, for example, run statistics, ROIs, version information of the firmware, hardware article and serial numbers as well as the start and end times of the measurement.
    • The visibility can be switched off and on again for one or more selected spectra. Each visible spectrum is given a filled circle in the explorer in the color of the displayed spectrum.
    • If a spectrum has been calibrated, this is shown in the circle to the left of the spectrum name. A "2" corresponds to a two-point calibration, a "1" stands for a one-point calibration and an uncalibrated spectrum does not get a number.
    • The spectrum and the associated file on the host system can be renamed via mouse right-click>>[Rename].
    • The spectrum can be deleted completely from the host system by mouse right-click>>[Delete], or pressing the delete key on the keyboard.

Interface Watch

The interface watch view offers the possibility of following all VICOLib functions executed by VICOScope. The maximum number of rows is limited to 1000.

Each line contains a time stamp according to ISO 8601 in the format YYYY-MM-DDTHH:MM:SS.SSS and the VICOLib function name used. A hover box is displayed shortly after moving the pointer over a specific line. In addition to the status (e.g. success or detailed error), the arguments used for the VICOLib function call are also noted. An error is indicated using a red background.

With the help of the [Clear] button, all lines are deleted and the interface watch is emptied.

Spectrum

A step function is used as a representation for the spectrum. The x-axis shows the energy in kiloelectron volts (Energy/keV). The events per channel are shown without unit of measurement on the y-axis (Events). If the spectrum is not calibrated, each channel has a width of 5eV. The center of the first channel is thus at 0eV and ranges from -2.5eV to +2.5eV. As soon as the spectrum is calibrated the channel width is adjusted according to the gain (see Analyze for more information). Up to 32 different colors are used to represent the spectra.

By clicking on [Start], the spectrum recording within the active project is started. The start button immediately changes to [Pause] to allow the spectra acquisition to be paused. Using [Resume], which is shown now, the recording can be continued. [Stop], which was initially greyed out, can now be used to stop the spectra recording. If the stop condition defined in the configuration view is reached, the recording is automatically stopped.

The switch can be used to toggle between a logarithmic and a linear representation.

The current cursor position within the chart is indicated in the upper right corner. It is displayed in the form of an ordered pair: (Energy/keV, Events).

As soon as an ROI has been selected within the analyze view and is no longer displayed transparently, you can move the ROI borders to the left and right within the chart with a mouse left-click (press and hold).

Oscilloscope

The oscilloscope view is used to acquire and display waveforms of internal digital pulse processor (DPP) signals for diagnostic purposes. Using the drop-down menu in the upper left the user can choose one of the following internal signals which can be acquired:

  • ADC-Data: Raw signal data of the ADC.
  • Fastfilter: Filter output of the fastfilter.
  • Mediumfilter: Filter output of the mediumfilter.
  • Slowfilter: Filter output of the slowfilter.
  • Baseline-average: Moving average of baseline mean value.
  • Baseline-sample: Latest acquired baseline values.

Furthermore, the user can set up the trigger condition and the sampling rate using the DPP parameters "Event Trigger Source", "Event Trigger Value", "Event Scope Sampling Interval", and "Event Scope Trigger Timeout" in the Configuration view.

Using the [Get Scope] button the user can initiate the acquisition of the selected waveform using the current parameter settings. The DPP now will wait for the trigger condition to occur and then read out the selected waveform. In case the trigger condition is met during the configured timeout time, the acquired waveform will be displayed. Otherwise, an error message will be shown.

On the x-axis of the plot the time in µs is shown. Depending on the chosen sampling rate VICOScope will automatically calculate the correct time values on the x-axis. The trigger moment is always at half of the waveform length (50% pre-trigger and 50% post-trigger). The y-axis shows digital signal values. Except for the ADC-data their absolute values have no practical use. For ADC-data however, the digital values are ADC codes where 0 corresponds to an analog voltage of -1V and 65535 corresponds to an analog voltage of +1V at the preamplifier input.

Using the [Export] button the user can save the displayed waveform into a JSON file. This is useful in case further analysis of the data with external tools is needed.

The plot of the displayed waveform supports the same user interactions as the Spectrum view (e.g., reset fit to view, scroll, zoom, ...).

Analyze

Within the analyze view up to 16 regions of interest (ROI) can be defined. There are two ways to create a new ROI. Either within the analyze view using a mouse right-click>>[Add ROI] or with a mouse/keyboard action within the spectrum view. To delete a ROI, you can either mark the desired row inside the analyze view and press the delete key on the keyboard or remove the ROI via mouse right-click>>[Delete ROI] within a row.

Measurement Series

With the help of the measurement series view, predefined measurement configurations, e.g. with different digital pulse processor (DPP) parameters, can be recorded automatically. This is a convenient way to optimize different parameters for a specific application.

  • Predefined measurement series configuration files in JSON format must be used. An example with all necessary information is shown below.
    • Parameters printed in italic are mandatory in order to be able to start a measurements series.
    • If a mandatory parameter is missing or misspelled, an error message is displayed.
  • It's possible to browse to the configuration file via [Set Configuration]. The path will be shown as configuration path.
  • The description can be used for documentation purposes. The content of the input field is saved in the output file.
  • The measurement series is started, all current ROIs are deleted and one ROI with the defined calibration energy is added by clicking on [Start]. The running configuration can be stopped via [Stop].
  • The current status is displayed above the estimated end time and the remaining time.
  • In the table at the bottom end a few important measurement results are shown.
  • A complete JSON output file (*.vms), which is updated periodically during the measurement, is saved within the project folder. It contains calculated analysis values, calibration information (e.g. ROI), run statistics, parameter settings and spectrum raw data.

Run Statistics

The statistics of the currently active spectrum are visible in the run statistics view.

  • Realtime/s: Time in seconds that has elapsed since the spectrum run started.
  • Livetime/s: Time in seconds in which the pulse detection was active during the spectrum run (pulse detection filter below trigger threshold and no reset detected).
  • IC: The total amount of events detected by the fastfilter is called input counts (IC).
  • OC: The total amount of successfully evaluated events is called output counts (OC).
  • ICR/cps: The input count rate (ICR) is an estimation of the X-ray photon rate in counts per second and calculated as input counts divided by livetime.
  • OCR/cps: The output count rate (OCR) indicates the rate of successfully evaluated events in counts per second and is calculated as output counts divided by the realtime.
  • Deadtime/%: The dead time ratio indicates the percentage of detected events that could not be evaluated (e.g., due to pile up or reset) and is calculated as 1 – OCR/ICR.

Configuration

The configuration view contains all digital pulse processor (DPP) parameters available to the end user. They are divided into four groups: I. Run Control, II. Pulse Processing, III. Hardware and Diagnostic, IV. Communication. Some of the parameters are only visible after toggling the switch from Essential to Advanced. This VICOScope help view features an individual description for each parameter, which is shown after clicking in the parameter row.

In order to set up the DPP in a suitable way for a certain application, each parameter value in the configuration view can be adjusted. Depending on the data type of the parameter the user can ether:

  • Select or de-select a checkbox for Boolean-type parameters.
  • Choose an entry using a drop-down menu for Enum-type parameters.
  • Type in a value for numerical-type parameters.

Since not every value is applicable for numerical-type parameters VICOScope might change the value entered by the user. VICOScope will find an applicable value which is as close as possible to the entered non-applicable value. In order to indicate that the user has entered a non-applicable value, the parameter will be highlighted with orange background color. As the value in the input box is changed to an applicable value in this case, a mouseover dialog informs the user which non-applicable was entered.

With the buttons [Save] and [Load] data transfer with the non-volatile parameter data memory of the DPP can be controlled:

  • Save: After changing a parameter within the configuration view, the new value is used for the next spectrum acquisition and is stored as the working copy in volatile memory inside the FPGA. Only after clicking on [Save], the value is saved non-volatile and will be used even after a power cycle.
  • Load: Non-overwritable default parameters that were written for the end user by KETEK upon delivery can be loaded via [Load]>>Default. The last saved parameters can be loaded via [Load]>>Saved. Unsaved changes must be discarded.

The user can find out for each parameter which value is currently stored within the non-volatile memory, by moving the pointer over the parameter entry. The displayed hover box shows the currently used parameter value, the parameter value stored in the default parameter set and the parameter value stored in the user parameter set. Furthermore, parameters are highlighted with yellow background color if the used parameter value does not match the saved value and parameters are highlighted with bold font, if the used parameter value does not match the default value.

Note: In the following, bin and channel are used interchangeably.

Parameter Name Parameter Description
Stop Condition Type The stop condition can be used to automatically stop MCA data acquisition when a certain criterion is met. This parameter configures the type of automatic stop criterion:
  • None: Disables automatic stop of MCA data acquisition.
  • StopAtFixedInputCounts: MCA data acquisition is stopped when a fixed number of input counts is reached.
  • StopAtFixedLivetime: MCA data acquisition is stopped when a fixed livetime is reached.
  • StopAtFixedOutputCounts: MCA data acquisition is stopped when a fixed number of output counts is reached.
  • StopAtFixedRealtime: MCA data acquisition is stopped when a fixed realtime is reached.
Stop Condition Value The stop condition can be used to automatically stop MCA data acquisition when a certain criterion is met. This parameter configures the value of automatic stop criterion:
  • If parameter "Stop Condition Type" is set to None: This parameter has no influence since automatic stop of MCA data acquisition is disabled.
  • If parameter "Stop Condition Type" is set to StopAtFixedLivetime or StopAtFixedRealtime: This parameter sets the livetime or realtime at which the MCA data acquisition is stopped.
  • If parameter "Stop Condition Type" is set to StopAtFixedInputCounts or StopAtFixedOutputCounts: This parameter sets the input or output counts at which the MCA data acquisition is stopped.
MCA Number Of Bins Adjust the number of bins used in the multichannel analysis (MCA).

This parameter determines the size of the spectrum by setting the number of bins/channels used in the MCA. At a higher number of bins, the spectrum is more fine-grained and a wider energy range can be fitted into the MCA. However, data size and statistical errors increase with a higher number of bins.
MCA Bytes Per Bin Adjust the number of bytes in each bin of the multichannel analysis (MCA).

This parameter determines the data depth used in each bin/channel of the MCA. A higher data depth allows the representation of larger values (e.g., 2^(3*8)-1 = 16777215 for 3 bytes per bin) before the bin overflows. However, data size increases with a higher number of bytes per bins.
Slowfilter Peaking Time Adjust the peaking time of the slowfilter.

Since the slowfilter is used for the determination of X-ray energies, this parameter has a major influence on energy resolution and signal throughput. Typically, a longer slowfilter peaking time improves the energy resolution at the cost of increased dead time.
Slowfilter Gap Time Adjust the gap time/flattop time of the slowfilter.

The gap time of the slowfilter is used to minimize ballistic deficit effects in the X-ray energy determination caused by the finite rise time of X-ray events. Long gap times offer an effective reduction of ballistic deficit effects (e.g., degradation of energy resolution, low-energy tail of peaks). However, long gap time increase the length of the slowfilter and therefore cause an increased dead time.
Mediumfilter Trigger Threshold Adjust the trigger threshold for pulse detection with the mediumfilter.

In case mediumfilter pulse detection is enabled, X-ray events are detected when the mediumfilter output exceeds this trigger threshold. Therefore, the mediumfilter trigger threshold should be chosen as low as possible in order to detect low-energy X-rays. However, at very low values the mediumfilter output will exceed the trigger threshold not only for X-ray events, but also for random noise. In this case a significant zero energy noise peak rises in the spectrum and dead time is increased.
Mediumfilter Pulse Detection Enable Enable or disable utilization of the mediumfilter for pulse detection.

If this parameter is enabled the mediumfilter will be used instead of the fastfilter for the detection of X-ray events. Typically, the mediumfilter has a higher length compared to the fastfilter and therefore the detection of low energy X-rays can be improved. However, pile up rejection worsens in this case.
Mediumfilter Maximum Width Adjust the maximum width of the mediumfilter.

The maximum width is a secondary pile up criterion using the duration/width of the mediumfilter output above the mediumfilter trigger threshold. A detected X-ray is rejected in case this duration is higher than the defined maximum width and when using the mediumfilter for pulse detection.

Choose a maximum width at least twice the mediumfilter peaking time expected the rise time of X-ray events. In order to disable this secondary pile up criterion choose a maximum width way above twice the mediumfilter peaking time plus maximum X-ray rise time. This parameter only has an influence if mediumfilter pulse detection is enabled.
Fastfilter Peaking Time Adjust the peaking time of the fastfilter.

Since the fastfilter is normally used for pulse detection, short peaking times can be used to improve pile up rejection. However, detection efficiency of low energy X-rays might suffer at short fastfilter peaking times.
Fastfilter Trigger Threshold Adjust the trigger threshold for pulse detection with the fastfilter.

X-ray events are detected when the fastfilter output exceeds this trigger threshold. Therefore, the fastfilter trigger threshold should be chosen as low as possible in order to detect low-energy X-rays. However, at very low values the fastfilter output will exceed the trigger threshold not only for X-ray events, but also for random noise. In this case a significant zero energy noise peak rises in the spectrum and dead time is increased.
Fastfilter Maximum Width Adjust the maximum width of the fastfilter.

The maximum width is a secondary pile up criterion using the duration/width of the fastfilter output above the fastfilter trigger threshold. A detected X-ray is rejected in case this duration is higher than the defined maximum width.

Choose a maximum width at least twice the fastfilter peaking time expected the rise time of X-ray events. In order to disable this secondary pile up criterion choose a maximum width way above twice the fastfilter peaking time plus maximum X-ray rise time.
Reset Inhibit Time Adjust the time duration for which the pulse processing is disabled after each detector reset.

Whenever a highly negative slope is detected in the detector signal the pulse processing is disabled in order to prevent a degradation of the spectrum due to detector resets. When the slope of the detection signal becomes positive again, pulse processing additionally stays disabled for the duration of the chosen reset inhibit time.

Therefore, reset inhibit time should exceed the time needed by the detector to settle into normal operation after the reset slope becomes positive again. However, long reset inhibit times lead to higher dead times, most significantly at high input count rates and high X-ray energies.
Baseline Average Length Adjust the number baseline samples used to calculate the running average of baseline value.

The baseline is the output of the slowfilter during the absence of X-ray events. A running average of the baseline is calculated using baseline samples acquired from the mediumfilter output in between X-ray events.

Higher baseline average lengths reduce the noise introduced by baseline correction and might improve the energy resolution. However, the reaction time to changes in the baseline (e.g., due to change in detector temperature) worsens at high baseline average lengths. In most cases a high baseline average is the best choice.
Baseline Trim Adjust the baseline trim setting which determines the length of the mediumfilter.

Long mediumfilter provide less noisy baseline samples and reduce the need of baseline averaging. However, long mediumfilter can acquire baseline samples less frequently than a short mediumfilter because they take longer to return to the baseline after an X-ray event. In most cases an intermediate mediumfilter is the best choice.
Baseline Correction Enable Enable or disable utilization of baseline correction.

The baseline is the output of the slowfilter during the absence of X-ray events. Main cause of a non-zero baseline is the presence of detector leakage current. When the baseline correction is enabled the current baseline average value is subtracted from each determined X-ray energy in order to eliminate the offset caused by detector leakage current. In most cases the utilization of the baseline correction is beneficial.
Digital Energy Gain Adjust the digital gain for energy values.

Every determined energy value will be multiplied with this gain. Therefore, large values will "stretch" and small values will "compress" the energy spectrum in the MCA data. Examples:

  • A digital energy gain value of 5461 can be used for an MCA bin width of approximately 5eV/bin.
  • A digital energy gain value of 4096 can be used for an MCA bin width of approximately 6.67eV/bin.
  • A digital energy gain value of 2730 can be used for an MCA bin width of approximately 10eV/bin.
  • A digital energy gain value of 1365 can be used for an MCA bin width of approximately 20eV/bin.
  • A digital energy gain value of 10922 can be used for an MCA bin width of approximately 2.5eV/bin.

Please note that the exact value of eV/bin for a certain digital energy gain depends on the specific analog gain of the detector and preamplifier and might slightly differ due to component variations.

Together with the parameter "MCA Number of Bins" the digital energy gain also defines the maximum energy that fits into the MCA. Example: When using a MCA with 8192 bins and a digital energy gain of 5461, the maximum energy in the MCA is approximately 5eV/bin * 8192 bins = 40960eV.
Digital Energy Offset Adjust the digital offset for energy values.

This offset it added to every determined energy value. A value of 32768 equals to zero offset, a value of 0 equals to the maximum negative offset of 256 bins and a value of 65535 equals to the maximum positive offset of 256 bins. This parameter can be used to implement a 2-point energy calibration of the MCA on the device. Otherwise, a value of 32768 (zero offset) is the best choice in most cases.
Dynamic Reset Enable Enable or disable utilization of dynamic reset.

When enabled the DPP triggers the detector reset when the detector signal reaches the voltage set by the parameter "Dynamic Reset Threshold" and no X-ray event is currently processed. In most cases the detection efficient for high energy X-rays can be improved by enabling the dynamic reset.
Dynamic Reset Threshold Adjust the threshold for the dynamic reset.

When the dynamic reset is enabled by the parameter "Dynamic Reset Enable" the DPP triggers the detector reset when the detector signal reaches the voltage set by this parameter. The threshold voltage is set as ADC Code. The minimum code 0 corresponds to a signal voltage of -1V and the maximum code 65535 corresponds to a signal voltage of +1V. Therefore, each code represents about 30µV.

A good practice to choose an appropriate threshold for the dynamic reset is to first determine the highest X-ray energy present in a certain application (e.g., excitation energy of 40keV). As a second step calculate the corresponding voltage of this energy using the KETEK default gain of 5mV/keV (e.g., 40keV correspond to 200mV) and convert to ADC codes of 30µV each (e.g., 200mV correspond to about 6667 ADC codes). Subtract the calculated amount of ADC codes from 65535 and choose a threshold for the dynamic reset about 2000 ADC codes lower than the result (e.g., subtraction of 6667 from 65565 gives 58898 and a good dynamic reset threshold would be about 57000). Using this method, a threshold for the dynamic reset can be found which still allows the detection of the highest X-ray energy prior to the detector reset.

Also, the dynamic reset threshold can be optimized empirically for a certain application. In this case variate the dynamic reset threshold until the value is found at which the maximum output count rate is achieved. In most cases a suitable value is found in the range of 50000 to 60000.
Dynamic Reset Duration Adjust the duration of the dynamic reset output pulse from the DPP to the preamplifier.

The duration of the reset pulse must be sufficiently long to be recognized by the preamplifier. In most cases a value of 200ns is a good choice.
Analog Hardware Power Down Enable or disable power down of DPP analog hardware.

In order to reduce power consumption in times no measurement is active the analog hardware of the DPP (e.g., ADC and operational amplifiers) can be set in power down. If this parameter is enabled the analog hardware is not powered.
Event Trigger Source Choose the trigger source used when reading out signal traces with the oscilloscope feature:
  • ADCOutOfRange: Trigger when an ADC overrange occurs (overvoltage or undervoltage at ADC signal input).
  • ActiveResetInitiate: Trigger when a dynamic reset is initiated by the DPP.
  • FastfilterPileup: Trigger when a pileup is rejected in the fastfilter using the maximum width criterion defined by the parameter "Fastfilter Maximum Width".
  • FastfilterResetDetected: Trigger when a reset is detected in the fastfilter.
  • FastfilterTrigger: Trigger when an input count is detected in the fastfilter.
  • InstantTrigger: Trigger is activated instantly without any condition.
  • MediumfilterPileup: Trigger when a pileup is rejected in the mediumfilter using the maximum width criterion defined by the parameter "Mediumfilter Maximum Width".
  • MediumfilterResetDetected: Trigger when a reset is detected in the mediumfilter.
  • MediumfilterTrigger: Trigger when an input count is detected in the mediumfilter.
  • NewBaselineSample: Trigger when a new baseline sample is acquired.
  • NewOutputCountAnyEnergy: Trigger when a new output count of any energy is added to the MCA.
  • NewOutputCountSpecificEnergy: Trigger when a new output count of a specific energy is added to the MCA. The specific MCA bin has to be defined using the parameter "Event Trigger Value".
  • SpecificADCValue: Trigger when an specific output value of the ADC is reached. The specific ADC code has to be defined using the parameter "Event Trigger Value".
Event Trigger Value Adjust the trigger value used when reading out signal traces with the oscilloscope feature.

This parameter is used to define the specific ADC value or specific MCA bin in case the parameter "Event Trigger Source" is set to SpecificADCValue or NewOutputCountSpecificEnergy. If any other value is used in parameter "Event Trigger Source" this parameter has no influence.
Event Scope Sampling Interval Adjust the sampling frequency used when reading out signal traces with the oscilloscope feature.

The signal traces have a fixed length of 8192 data points. By adjusting the sampling frequency using this parameter the number of signal samples obtained in one second can be chosen. Therefore, a suitable time base for the desired signal can be set up.

The value of this parameter represents the factor of down-sampling regarding the 80MHz system clock. Examples:

  • Setting this parameter to a value of 1 results in a sampling frequency of 80MHz/1=80MHz for the signal trace. The 8192 data points represent a time frame of 8192/80MHz=102.4µs.
  • Setting this parameter to a value of 2 results in a sampling frequency of 80MHz/2=40MHz for the signal trace. The 8192 data points represent a time frame of 8192/40MHz=204.8µs.
  • Setting this parameter to a value of 10 results in a sampling frequency of 80MHz/10=8MHz for the signal trace. The 8192 data points represent a time frame of 8192/8MHz=1.024ms.
  • Setting this parameter to a value of 100 results in a sampling frequency of 80MHz/100=800kHz for the signal trace. The 8192 data points represent a time frame of 8192/80kHz=10.24ms.
  • ...

Please note that data collection for signal traces can take a considerable amount of time when using very low sampling frequencies.
Event Scope Trigger Timeout Adjust the maximum waiting time for a successful trigger when reading out signal traces with the oscilloscope feature.

In case the trigger condition set up by the parameters "Event Trigger Source" and "Event Trigger Value" was not met during this time, a timeout error will be returned. Choosing a high timeout value is useful in cases where the trigger condition rarely occurs and a long maximum waiting time for the response (e.g., 16 seconds) is acceptable for the user.
SPI Power Down Enable or disable power down of SPI interface.

In case no SPI communication is needed (e.g., ethernet interface of is used) the SPI interface can be set in power down. If this parameter is enabled the device is not accessible via SPI. SPI power down has practically no impact on power consumption. However, unintentional SPI communication can be prevented by setting the interface to power down.
USB Power Down Enable or disable power down of USB hardware.

In order to reduce power consumption when no USB communication is needed (e.g., ethernet interface of is used) the USB interface can be set in power down. If this parameter is enabled the USB hardware is not powered.
Ethernet Power Down Enable or disable power down of ethernet hardware.

In order to reduce power consumption when no ethernet communication is needed (e.g., USB interface of is used) the ethernet interface can be set in power down. If this parameter is enabled the ethernet hardware is not powered.
Ethernet Apply The [Apply] button will be activated after changing one of the parameters "Ethernet Protocol", "Ethernet Speed", "Ethernet IP Address, "Ethernet subnet mask" or "Ethernet gateway IP address". In this case use the [Apply] button for the changes of these parameters to become effective or use the "Cancel" button in order to discard the changes of these parameter.

Please note that after using the [Apply] button any ethernet connection with the device will be gone since the ethernet interface is reconfigured. Therefore, a new scan is needed after using the [Apply] button in case you are connected via ethernet to the device.
Ethernet Protocol Choose the transport layer protocol used for ethernet communication. Supported protocols are:
  • TCP: Transmission Control Protocol
  • UDP: User Datagram Protocol

Please be aware to use the [Apply] button to ensure the changes to this parameter take effect. Additionally, a rescan may be necessary in case of a connection loss.
Ethernet Speed Choose speed and duplex mode ethernet communication. Options are:
  • AutoNegotiation: Use auto-negotiation to determine ethernet speed and duplex mode.
  • FullDuplex100MBits: Force ethernet to run at 10Mbit/s and full duplex mode.
  • FullDuplex10MBits: Force ethernet to run at 10Mbit/s and full duplex mode.
  • HalfDuplex100MBits: Force ethernet to run at 100Mbit/s and half duplex mode.
  • HalfDuplex10MBits: Force ethernet to run at 10Mbit/s and half duplex mode.

In most cases the auto-negotiation options are suitable. The ethernet interface will automatically use the highest performance speed and duplex mode supported by each device present in the network.

Forcing a specific ethernet speed using another option than auto-negotiation can cause the communication to fail (e.g., due to network switches that do not support the chosen option). However, not using auto-negotiation might be useful in case power consumption is critical since 10Mbit/s options draw significantly less power in comparison to 100Mbit/s options.

Please be aware to use the [Apply] button to ensure the changes to this parameter take effect. Additionally, a rescan may be necessary in case of a connection loss.
Ethernet IP Address Choose the IP address used by the DPP ethernet interface. Internet Protocol version 4 (IPv4) standard is used.

Please be aware to use the [Apply] button to ensure the changes to this parameter take effect. Additionally, a rescan may be necessary in case of a connection loss.

Please be aware to use the [Apply] button to ensure the changes to this parameter take effect. Additionally, a rescan may be necessary in case of a connection loss.
Ethernet Gateway IP Address Choose the gateway IP address used by the DPP ethernet interface. Internet Protocol version 4 (IPv4) standard is used.

Please be aware to use the [Apply] button to ensure the changes to this parameter take effect. Additionally, a rescan may be necessary in case of a connection loss.
